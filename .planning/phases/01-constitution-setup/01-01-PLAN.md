Cover these 8 rules in 6 articles (per D-06: 6 articles, each with rule + justification + consequence):

**Article 1 — Margin of Error Mandatory on All Sample-Based Claims:**
  Rule: Every percentage or proportion derived from a sample MUST be accompanied by margin of error at 95% confidence level (z_crit = 1.96).
  Provide formula: MoE = z_crit × √(p(1−p)/n).
  Consequence: Critic refuses to pass output to Designer until MoE is computed and displayed.

**Article 2 — Statistical Significance Threshold (p < 0.05):**
  Rule: All claims of difference, correlation, or effect MUST report p-value. p ≥ 0.05 disallows "significant" language — say "no statistically significant difference found."
  Consequence: Critic downgrades unsupported significance claims and appends caveat.

**Article 3 — Prohibition of Prose Fluff:**
  Rule: Output must go straight to data. Forbidden phrases include: "It's important to note that...", "Based on the data provided...", "Interestingly...", "It is worth mentioning...", "One can observe that...", "The data suggests that...", "In conclusion...", "As we can see...".
  Detection: Deletion test — if a sentence can be removed without losing a number, named entity, or finding, it is fluff. Delete it.
  Consequence: Critic scans output for these phrases before delivery. Any match → return to Designer for removal.

**Article 4 — Prohibition of Percentages in Qualitative Samples (N < 30):**
  Rule: When N < 30 for qualitative data (open-ended responses, interviews, focus groups), NEVER report percentages. Quote verbatims directly; note frequency as "mencionado por n de N participantes" without false precision.
  Consequence: Critic flags any quantitative generalization from small qualitative samples. Blocks delivery.

**Article 5 — Minimum Sample Size for Parametric Tests:**
  Rule: Parametric tests (t-test, ANOVA, Pearson r, linear regression) require N ≥ 30 per group. Below threshold: use non-parametric equivalents (Mann-Whitney U, Kruskal-Wallis, Spearman ρ).
  Consequence: Statistician must flag when parametric assumptions violated. Critic enforces and recommends non-parametric alternative.

**Article 6 — Data Quality Gates:**
  Combines all 3 data quality rules from D-05:
  6a. Straight-lining detection: ≥80% identical consecutive Likert-scale responses in a single respondent → flag as potentially inattentive. Statistician computes straight-lining rate per variable.
  6b. Percentage sum validation: All percentage columns across mutually exclusive categories MUST sum to 100% (±0.5% rounding tolerance). Any deviation → flag.
  6c. Missing data threshold: Any variable with >10% missing responses → flag. Critic must report missingness rate and assess potential bias direction.

Format guidelines per D-06 and RESEARCH.md template:
- Use `## Article {N} — {Title}` as heading
- Use `### Rule`, `### Justification`, `### Violation Consequence`, `### Detection Method / Computation` as subheadings

- Include a preamble explaining that constitution.md is read by the Critic agent during every analysis command

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Raw data input → Statistician | Untrusted survey data enters the system — may contain errors, missing values, straight-lining, or fabricated numbers |
| Statistician output → Critic | Validated metrics cross the trust boundary — Critic must independently verify, not trust Statistician blindly |
| Critic output → Tufte Designer | Audit findings must be faithfully incorporated — Designer cannot ignore or soften Critic flags |
| Constitution.md rules → Critic enforcement | Rules must be mechanically enforceable — vague rules cannot be enforced |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | AI inference (hallucinated statistics) | mitigate | Statistician validates all numbers against input data; Critic flags unsupported claims; constitution.md Article 1 requires MoE on every percentage |
| T-01-02 | Tampering | AI inference (confirmation bias) | mitigate | Critic explicitly audits for confirmation bias (audit dimension 1 in agent-critic.md); SKILL.md Stage 2 instructions require checking "does analysis only test hypotheses the user favors?" |
| T-01-03 | Information Disclosure | AI output (overgeneralization from small samples) | mitigate | constitution.md Article 4 prohibits percentages for N<30 qualitative; Critic flags and blocks delivery; constitution.md Article 5 restricts parametric tests to N≥30 |
| T-01-04 | Tampering | AI output (prose fluff masking weak findings) | mitigate | constitution.md Article 3 lists forbidden phrases + deletion test; Critic scans output before delivery; SKILL.md Tufte rules reiterate enforcement |
| T-01-05 | Repudiation | AI output (spurious correlation claims) | mitigate | constitution.md Article 2 requires p<0.05; Critic checks for confounding variables and plausible causal mechanisms |
| T-01-06 | Elevation of Privilege | AI output (conclusions from incomplete data) | mitigate | constitution.md Article 6 flags missing >10% and straight-lining; Critic reports missingness patterns and potential bias direction |
| T-01-SC | Tampering | N/A — no packages installed | accept | Phase 1 has zero external package dependencies. RESEARCH.md Package Legitimacy Audit confirms: pure Markdown + YAML, no npm/pip/cargo installs. |
