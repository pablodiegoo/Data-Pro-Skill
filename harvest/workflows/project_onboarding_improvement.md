# Proposed Improvement: Project Onboarding

## Context
Project onboarding currently generates rules for coding, data governance, and QA, but it lacks strict reporting guidelines.

## Proposal
- **Phase 3 (Automated Governance)**: Add the automated creation of `reporting-standards.md` to enforce the 100% Markdown (zero external images) rule with Mermaid.js.
- **Phase 4 (Strategy & Roadmap)**: Expand the "Project Brief" step to mandate the inclusion of the two-tier reporting scripts (EDA first, Final Report second) in the initial task mapping.
